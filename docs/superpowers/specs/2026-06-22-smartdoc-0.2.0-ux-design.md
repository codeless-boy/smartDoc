# smartDoc 0.2.0 — UI/UX 打磨 设计文档

> 日期: 2026-06-22 | 版本: 0.2.0 | 状态: 已确认

## 1. 目标与范围

在 0.1.0 三栏框架（顶部搜索 + 左侧筛选 + 中间列表 + 右侧抽屉）保持不变的前提下，对 UI/UX 做一次系统性打磨。本次升级不引入新功能模块（不做批量操作、不做预览、不做暗色），专注于：

- **视觉风格**：从 antd 默认主题切换到"干净克制"中性灰阶 + 单点墨色强调，通过 antd `ConfigProvider` 的 theme token 集中管理。
- **交互打磨**：补齐空状态/加载态/无结果三种缺失场景；引入键盘快捷键；引入文件行右键上下文菜单。
- **环境清理**：移除 Electron 默认菜单条（File / Edit / View / Window / Help），仅在 dev 模式保留 DevTools 快捷键。

**明确不做**：批量选择、暗色主题、布局结构调整（侧栏宽度/抽屉常驻/卡片网格等）、新功能模块。

## 2. 主题 Token 体系

新建 `src/renderer/src/theme.ts` 作为唯一配色事实来源；`App.tsx` 把它传给 `<ConfigProvider theme={...}>`，antd 组件全部继承。组件内联 style 里硬编码的色值改为引用 theme 导出的常量，或直接删除让 antd token 接管。

```ts
// src/renderer/src/theme.ts
import type { ThemeConfig } from 'antd'

export const theme: ThemeConfig = {
  token: {
    colorPrimary: '#1f1f1f',         // 强调色：墨色（按钮/选中态/链接）
    colorBgLayout: '#f7f8fa',         // 页面底
    colorBgContainer: '#ffffff',      // 卡片/表格行底
    colorBorder: '#e8e8e8',           // 细而淡的分割线
    colorBorderSecondary: '#f0f0f0',  // 更淡的次级分割线
    colorText: '#1f1f1f',
    colorTextSecondary: '#595959',
    colorTextTertiary: '#8c8c8c',
    borderRadius: 6,
    fontSize: 13,                     // 比默认 14 略小，更紧凑专业
    controlHeight: 30
  },
  components: {
    Table: {
      headerBg: '#fafafa',
      headerSplitColor: 'transparent',
      rowHoverBg: '#f5f5f5'
    },
    Layout: { siderBg: '#ffffff', headerBg: '#ffffff' },
    Button: { primaryShadow: 'none' },
    Tag: { defaultBg: '#f5f5f5' }
  }
}
```

App.tsx 改造：

```tsx
import { ConfigProvider } from 'antd'
import zhCN from 'antd/locale/zh_CN'
import { theme } from './theme'
import { AppShell } from './components/AppShell'

export function App(): JSX.Element {
  return (
    <ConfigProvider locale={zhCN} theme={theme}>
      <AppShell />
    </ConfigProvider>
  )
}
```

**配色合理性**：墨色 `#1f1f1f` 作主色而非默认 antd 蓝（`#1677ff`）——避免"任何用 antd 的产品都像 antd 范例"，且与 0.1.0 的 indigo `#6366f1` 也做了切割，因为后者偏品牌、过于醒目；中性墨色更耐看、长时间用不疲劳。

## 4. 去除 Electron 默认菜单条

**问题**：当前 `src/main/index.ts` 没有自定义菜单，Electron 默认显示 File / Edit / View / Window / Help 菜单条，其中的项（Reload、Toggle DevTools、Force Reload 等）与 smartDoc 无关，反而干扰用户。

**做法**：在 `app.whenReady()` 内、`createWindow()` 之前调用 `Menu.setApplicationMenu(null)`，移除所有窗口的应用菜单。

```ts
// src/main/index.ts
import { app, BrowserWindow, Menu } from 'electron'
// …
app.whenReady().then(async () => {
  try {
    Menu.setApplicationMenu(null)
    await bootstrap()
    createWindow()
    if (updater) updater.init(() => mainWindow)
  } catch (err) {
    logger.error('bootstrap failed', err)
    app.quit()
  }
  // ...activate handler
})
```

**副作用与对策**：

| 副作用 | 现状 | 对策 |
|---|---|---|
| F12 / Ctrl+Shift+I 打开 DevTools | 由默认菜单提供 | 仅 dev 模式（`!app.isPackaged`）下，在 `createWindow()` 内给 `mainWindow.webContents` 注册 `before-input-event` 监听，处理这两个组合键。生产构建不注册。 |
| Ctrl+C/V/X/Z 编辑快捷键 | 由 Chromium 自带（不依赖菜单） | 不受影响 |
| F11 全屏、Alt+F4 关闭等 | 由 BrowserWindow / 系统提供 | 不受影响 |

DevTools 快捷键注册：

```ts
// createWindow() 末尾追加
if (!app.isPackaged && mainWindow) {
  mainWindow.webContents.on('before-input-event', (event, input) => {
    const isF12 = input.key === 'F12'
    const isCtrlShiftI = input.control && input.shift && input.key.toLowerCase() === 'i'
    if (isF12 || isCtrlShiftI) {
      mainWindow?.webContents.toggleDevTools()
      event.preventDefault()
    }
  })
}
```

## 5. 空状态 / 加载态 / 无结果

新建 `src/renderer/src/components/EmptyState.tsx` 单一组件，按 `kind` 参数渲染三种场景。所有逻辑放进 FileTable.tsx 的渲染分支，根据 `files`/`loading`/`filter` 选择渲染表格或 EmptyState。

### 5.1 三个场景的判定

| 场景 | 触发条件 | 渲染 |
|---|---|---|
| 首次零文件 | `files.length === 0 && !loading && filter 全空` | `<EmptyState kind="onboarding" />` |
| 加载中 | `loading === true` | 骨架屏（6 行灰条） |
| 筛选无结果 | `files.length === 0 && !loading && filter 非空` | `<EmptyState kind="no-match" />` |

"filter 全空" = `!filter.keyword && !filter.tagIds?.length && !filter.exts?.length && !filter.untagged && !filter.topOpenedLimit`。

### 5.2 EmptyState 组件接口

```tsx
interface Props {
  kind: 'onboarding' | 'no-match'
  // onboarding 用：触发导入流程（与 TopBar 的 pickAndImport 共享同一函数）
  onImport?: () => void
  // no-match 用：当前激活的筛选条件描述（用于副标题展示）
  activeFilters?: string[]
}
```

**onboarding**：antd `<Empty image={Empty.PRESENTED_IMAGE_SIMPLE}>` + 标题 "还没有文件" + 副标题 "把文件拖进来，或点下方按钮选择" + primary 按钮 "导入第一个文件"。

**no-match**：同款 `<Empty>` + 标题 "没有匹配的文件" + 副标题列出当前激活的筛选（如 "标签: work, 类型: pdf"）+ 链接 "清除筛选"（调 `useAppStore.getState().resetFilter()`）。

### 5.3 加载态骨架屏

替代 antd Table 自带的转圈 spinner。用 antd `<Skeleton.Node>` 或自绘 6 行（与表格列对齐：图标格 24×24 灰块、文件名 60% 宽灰条、大小 80px 灰条、日期 80px 灰条）。

实测 list 查询通常 < 10ms，骨架屏可能闪一下即消失——这仍是正确的占位手段，避免任何"白屏 → 内容"的跳变。

### 5.4 共享导入入口

当前 `pickAndImport` 嵌在 TopBar 组件内部。重构：把它提到 AppShell 或单独的 hook `use-import.ts`，TopBar 按钮和 EmptyState 的 "导入第一个文件" 按钮共享同一函数。同时也为第 6 节键盘快捷键 Ctrl+I 服务。

## 6. 键盘快捷键

新建 `src/renderer/src/hooks/use-shortcuts.ts`，在 `AppShell` 调用一次，集中管理所有全局键盘事件。

### 6.1 快捷键清单

| 按键 | 行为 | 触发条件 |
|---|---|---|
| `Ctrl+F` / `/` | 聚焦顶部搜索框 | 任意；阻止浏览器默认 Ctrl+F |
| `Esc` | 关抽屉 → 清搜索 → 重置筛选（三级回退） | 详情抽屉打开 / 搜索框非空 / 有筛选 |
| `Delete` | 删除当前选中文件（带 antd Modal.confirm） | `selectedId` 非空 |
| `Ctrl+I` | 触发导入流程 | 任意 |
| `Enter` | 打开当前选中文件 | `selectedId` 非空 |
| `↑` / `↓` | 在文件列表上下移动选中行 | 列表区域有焦点；抽屉跟随更新 |

### 6.2 实现要点

- `useEffect` + `window.addEventListener('keydown', handler)`，组件卸载时移除。
- 输入框/textarea 内按键 **不触发**全局快捷键：检查 `e.target` 是否为 `INPUT` / `TEXTAREA` / `contentEditable`。`Esc` 例外——任何情况都响应。
- `Esc` 的三级回退：先判 `selectedId` → 再判 `keyword` → 再判 filter；只触发最优先匹配的那一级。
- 上下键：从 `useAppStore.getState()` 取 `files` 和 `selectedId`，计算当前 index，加减 1 后 clamp 到 `[0, files.length-1]`，调 `select(files[newIndex].id)`。
- `Delete` 用 antd `Modal.confirm` 弹确认（"删除「文件名.ext」？此操作会同时移除磁盘文件"）；确认后 `delete([id])` + `select(null)` + `refreshAll()`。

### 6.3 搜索框聚焦的实现

最简单的做法：TopBar 给搜索框加 `data-testid="search-input"`（已为 E2E 加过），shortcut hook 用 `document.querySelector<HTMLInputElement>('[data-testid="search-input"]')?.focus()`。无需 ref 链路，无需 forwardRef 重构。

### 6.4 文档化

README 底部新增"键盘快捷键"小节，列出全部 6 个组合。

## 7. 右键上下文菜单

新建 `src/renderer/src/components/FileContextMenu.tsx`。FileTable 行右键时弹出 antd Dropdown 菜单。

### 7.1 菜单项

| 项 | 行为 | 实现 |
|---|---|---|
| 打开文件 | 系统默认程序打开 | `window.api.file.open(id)` |
| 在文件夹中定位 | 资源管理器中选中 | 重新引入 `file:showInDir` IPC |
| —— 分割线 —— | | |
| 复制文件名 | 写剪贴板 | `navigator.clipboard.writeText(file.name)` |
| 复制路径 | 仓库内绝对路径写剪贴板 | 新增 `file.getAbsolutePath(id)` IPC |
| —— 分割线 —— | | |
| 编辑标签 | 打开抽屉聚焦标签段 | `select(id)`（抽屉本就以标签段为最顶部，无需额外滚动） |
| —— 分割线 —— | | |
| 删除 | antd Popconfirm 二次确认 | `delete([id])` + `refreshAll` |

### 7.2 触发与定位

- FileTable 的 `onRow` 增加 `onContextMenu(e)`：阻止默认 → 设置 store 里的 `contextMenuTarget: { id, x, y } | null`。
- AppShell 渲染 `<FileContextMenu />`：从 store 读 `contextMenuTarget`；若非 null，用 antd `Dropdown` 受控显示，位置由 `getPopupContainer` 配合一个 fixed 定位的 1×1 锚点 div 实现（锚点设 `style={{ position: 'fixed', left: x, top: y }}`）。
- 关闭条件：点击菜单外、按 Esc、滚动表格、表格 dataSource 变化。

### 7.3 store 字段扩展

`useAppStore` 新增：

```ts
contextMenuTarget: { id: string; x: number; y: number } | null
openContextMenu: (target: { id: string; x: number; y: number }) => void
closeContextMenu: () => void
```

### 7.4 复活 `file:showInDir`

0.1.x 阶段曾移除该 IPC（详情抽屉的"定位文件"按钮改成了"打开文件"，移除了唯一调用点）。0.2.0 因右键菜单需要重新引入：

- `src/shared/ipc-channels.ts`: 重新加入 `FileShowInDir: 'file:showInDir'`
- `src/main/ipc/file-ipc.ts`: 重新注册 handler，调 `shell.showItemInFolder(path.join(root, file.storagePath))`
- `src/preload/index.ts`: 在 `file` 命名空间内重新暴露 `showInDir(id)`

提交信息中明确说明此次"复活"避免代码考古时困惑。

### 7.5 新增 `file.getAbsolutePath` IPC

当前 `FileInfo.storagePath` 是仓库内相对路径（如 `files/<uuid>/<name>`）。复制路径需要绝对路径，仓库根在主进程。

- `src/shared/ipc-channels.ts`: 新增 `FileGetAbsolutePath: 'file:getAbsolutePath'`
- `src/main/ipc/file-ipc.ts`: 注册 handler，从 `repoRoot()` 与 `svc.list({filter:{}}).find(...)` 拼接绝对路径
- `src/preload/index.ts`: `file.getAbsolutePath(id): Promise<string | null>`

## 9. 技术选型与不做的事

### 9.1 选用

- **antd theme token** 而非全局 CSS 变量：与 antd 主题机制原生融合，组件自动继承，无需双系统维护。
- **antd Empty + 自绘骨架** 而非引入第三方占位库：减少依赖。
- **`document.querySelector` + `data-testid`** 而非 ref 链路传 input：与 E2E 选择器复用，无需 forwardRef 重构。
- **antd Dropdown 受控 + 1×1 锚点 div** 而非自研右键菜单：避免 z-index/portal/键盘焦点等坑。

### 9.2 明确不做

- 暗色主题：未来版本单独评估。
- 批量选择/批量操作：未来版本评估，本次不引入 `selectedRowKeys` 状态。
- 抽屉常驻：本次保持"点选才弹出"的现状。
- 卡片网格视图：本次保持紧凑列表。
- 文件预览（PDF / 图片缩略图）：未来版本评估。
- 设计文档 §8 边界场景（磁盘空间不足回滚、SQLite 损坏 recover）：与 UI/UX 不直接相关，未来版本评估。
- 设计文档 §4 提到的 MD5 字段：与 UI/UX 不直接相关，未来版本评估。
